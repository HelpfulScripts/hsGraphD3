  17 * `hsGraphD3` provides convienent programmatic shortcuts to plotting data with [`D3js`](https://d3js.org/).

  18 * 

  19 * 

  20 * ## Usage

  21 * 1. create a {@link Graph.Graph `Graph`} object with a root DOM element to attach to: 

  22 * `graph = new hsGraphD3.Graph(root);`

  23 * 2. add desired series configurations, specifying the type of series, as well as the data to use for the x- and y-axes, 

  24 * and other variables, depending on the series type:

  25 * `graph.add('area', {x:'time', y:'volume', r:()=>5});`

  26 * 3. apply any desired formatting changes to the default:

  27 * `graph.axes.defaults.color = '#88f';`

  28 * 4. render the graph for a given data set: 

  29 * `graph.render(data);`

  30 * 

  31 * To dynamically update the graph to new data, call the {@link Graph.RenderChain `update`} method.

  74 * ## Integrations

  75 * Besides using the standalone version (see examples above), `hsGraph` supports integration 

  76 * with [`Mithriljs`](http://mithriljs.org):

  77 * ```

  78 * m(Graph, {

  79 *      rootID: 'root',

  80 *      define: (graph:Graph) => {

  81 *          graph.add('column', {x:'date', y:'volume'});

  82 *          graph.defaults.scales.margin.left = 50;

  83 *      },

  84 *      data: data

  85 * })

  86 * ```  

  87 * Mithril will take care of the rendering automatically. Render updates can be done via two additional attributes, 

  88 * `updatePeriod` and `updateCallback`. See the {@link Graph.Graph.constructor Graph.constructor} for details:

  89 * 

  90 * ## Dynamically changing the data series

  91 * When defining the series (e.g. `graph.add('column', {x:'date', y:'volume'});`) 

  92 * the column names used as reference to the data are made available in the defaults of the series and can 

  93 * be dynamically changed there:

  94 * ```

  95 *    graph.defaults.series[0].dims.y = 'costs';

  96 * ```

  97 * If this is done within an event listener that Mithril is aware of, redrawing happens automatically. 

  98 * Otherwise a redraw can be triggered by calling `m.redraw();`
